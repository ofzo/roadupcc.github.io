规则说明 ：

标有👍的视为本次重构必须遵循的规则

文件规范

👍 文件名统一使用驼峰命名法, 有特殊含义的文件可使用 . 分割，如测试文件 a.test.js

    命名原则的指导思想是使得你自己和工作组的每一个成员能够方便的理解每一个文件的意义
    同类文件放在同一文件夹下

👍 项目结构，见目录结构
👍 静态资源的引入需经过webpack打包
👍 公共资源可以在html文件中直接链接, 可以不加 hsah ，以利用浏览器缓存, 可以部署到cdn上

开发环境

👍 NODE_ENV 用于定义build的模式，（development, production）
👍 GENERATOR_SOURCEMAP 用于定义是否开启 source-map 在本地环境、开发环境下开启
👍 条件编译使用

if (process.env.NODE_ENV === "development"){
  // 仅在开发环境下需要搞的事情，在production环境下，不会出现这部分代码
  // ...
}

👍 生产环境代码需压缩 — uglify
生产环境下，原则上不允许出现alert，console等日志，可配置uglify自动drop
👍 应用缓存 对于直接引用第三方类库，可配置缓存 需要nginx配合
代码分片 使用 import() 动态导入

Html

 原则上遵守w3c规范，避免通篇div+span， 尽可能兼顾 seo、语义化

👍 标签名全部使用小写字母
👍 标签必须有正确嵌套的闭合标签， 或者使用自闭合标签
👍 必须有 title 标签
👍 在 head 中引入css文件

👍 doctype 统一使用

<!DOCTYPE html>

👍 编码

<meta charset="utf-8" />

👍 meta 信息

<meta name="renderer" content="webkit|ie-comp|ie-stand" />
<meta http-equiv="X-UA-Compatible" content="IE=edge" />
<meta name="viewport" content="width=device-width,initial-scale=1,user-scalable=no, maxium-scale=1, minium-scale=1" />
<meta name="keywords" content="关键词,5个左右,单个8汉字以内" />
<meta name="description" content="网站描述，字数尽量空制在80个汉字，160个字符以内！" />
<meta http-equiv="Cache-Control" content="no-cache, must-revalidate" />

👍 html 语言类型

<html lang="zh-CN" />

👍 css样式表引入， 必须添加 rel=“stylesheet” 明确指明类型

<link rel="stylesheet" src="page.css" />

👍 favicon

<link rel="shortcut icon" href="favicon.ico" />

👍 JavaScript 在body后引入 — polyfill 需在head中引入

<script type=“text/javascript” src="polyfill.js"></script>

css

 避免使用计算属性值 cacl

👍 禁止使用 !important
👍 避免使用过多的 web font
尽量避免使用z-index调整层级，z-index 的值不要超过150
👍 配置 autoprefixer 自动添加所需要的前缀
👍 选择器层级不超过3层, 方便阅读同时可以提高浏览器的渲染性能 — 非强制要求
👍 推荐使用 sass 预处理 css
👍 规则排序 vs-code 安装 CSS comb 插件可自动排序, "``csscomb.preset"=yandex

1. 定位信息/Positioning
2. 展示类型及文档流相关/Display & Flow
3. 元素尺寸/Dimensions
4. 边距边框/Margins, Padding, Borders, Outline
5. 文字排版/Typographic Styles
6. 背景/Backgrounds
7. 其他如透明度，光标样式及衍生内容/Opacity, Cursors, Generated Content

javascript

👍 编程思想： OOP / Functional
👍 必须通过eslint的格式校验
同样的逻辑不写第二遍
大数据或耗时较长的请求，可配置localStorage
👍 使用fetch发起请求， 使用cookie时，需添加option，credentials = “some-origin”

React

https://www.npmjs.com/package/eslint-plugin-react

"plugin:react/recommended"
  • 文件

👍 组件名使用 PascalCase 命名法
👍 组件文件统一使用 .jsx或 .tsx 后缀名，文件名与default的组件同名

  • 组件

👍 核心原则：高内聚、低耦合
组件分 container 和 pure-component, container 包含业务逻辑, pure-component 只做内容展示
每个文件只包含一个类组件，可以包含多个 stateless component 或挂载在类组件的静态属性上, 尽量
使用 PureComponent
不要使用 mixins， 使用Higher-Order Components
👍 相关组件写在同目录的其他文件中
👍 始终使用 JSX 语法，禁止使用 React.createElement
👍 尽可能使用stateless component
👍 组件必须命名, 禁止直接作为default导出
简短的输出在行内直接三元运算符，运算逻辑简单的直接使用行内迭代
👍 结构复杂的组件需要拆分
减少不必要的组件层级，React.Fragment， 可简写成<></>
👍 禁止使用React.createClass定义组件
不要接收 ReactDOM.render 的返回值
👍 state 保持结构简单, 避免深层级结构
👍 受控组件需对外开放控制权限，默认不受限
尽量避免在组件内联注释
👍 对外开放的组件必须描述 props 结构，并有注释
禁止在componentDidMount 和 componentDidUpdate 中使用 setState
👍 禁止直接对 state 赋值
👍 不要使用字符串形式的 ref, 高版本中使用React.createRef, 尽量避免直接接触 DOM
👍 组件的 render 函数必须有返回值
👍 包含合成事件以外的事件监听, 必须在 componetWillUnmount 中移除 放置内存泄漏
使用 Readme.md编写示例, react-styleguide 可直接生成文档, jsdoc 或 type 用于开发环境下的 intellisense https://react-styleguidist.js.org/docs/getting-started.html

  • 属性

👍 传递给 HTML 元素的自定义属性，需要添加 data- 前缀，React 不会渲染非标准属性
👍 组件必须声明 propTypes， 需为非必须属性指明默认值 defaultProps
👍 属性值为 true 时，可以省略值
组件需接受样式注入，推荐以className属性的方式
属性较少时可以行内排列， 较多时属性单独成行
无子组件时使用自闭合标签
👍 避免使用index作为key， key应是可以唯一确定这条数据的
👍 禁止将setState、forceUpdate、replaceState等更新方法作为属性传递
👍 禁止通过 props 传递 this

  • 方法

👍 lifecycle 按React的调用顺序排序，并集中
👍 事件响应以on开头，以 Handler 结尾，使用箭头函数定义 例如 onItemClickHandler=(event)=>{}
👍 尽可能少用箭头函数定义方法
👍 禁用dangerouslySetInnerHTML — 特殊需求除外（富文本编辑器）
